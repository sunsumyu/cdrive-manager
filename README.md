# C 盘空间管理器

一个用 Rust 编写的桌面 GUI 磁盘空间分析工具，参考了 WinDirStat、QDirStat、Filelight、ncdu、Diskonaut、duc 和 BleachBit 的设计思路。

当前版本是安全优先的 MVP：**只扫描和可视化，不删除任何文件**。

## 功能

- 默认扫描 `C:\`，也可以输入其他目录。
- 支持系统目录选择窗口选择扫描目录。
- 开始扫描前会校验路径是否存在、是否是目录。
- 后台线程扫描，界面不会因为扫描 C 盘而完全卡死。
- 支持取消正在进行的扫描，并保留已扫描到的部分结果。
- 统计：
  - 累计大小
  - 文件数量
  - 目录数量
  - 访问错误数量
- 显示：
  - 最大目录
  - 最大文件
  - 文件扩展名/类型占用
  - 访问错误列表
- Treemap 矩形空间占用图，参考 WinDirStat / QDirStat。
  - 扫描完成或取消后基于完整目录树索引显示目录层级。
  - 支持点击目录块进入子目录视图。
  - 支持返回上级和返回根目录。
  - 支持显示当前目录“直属文件”空间块。
  - 支持在目录块右键复制路径或打开位置。
- 支持保存和打开 JSON 扫描结果缓存，重新打开后可继续查看完整目录树和 Treemap。
- 支持导出 CSV 报告，便于用表格工具分析概览、目录、最大文件、文件类型和错误记录。
- 支持 dry-run 清理预览：
  - 独立后台线程只读取文件元数据，不删除、移动或修改文件。
  - 可取消预览，并保留已发现的候选。
  - 初始保守规则包括临时文件扩展名、日志/转储、备份/旧文件和明显临时/缓存目录中的文件。
  - 会标记系统目录和常见用户文档目录等受保护候选，受保护候选不计入预计可清理空间。
  - 支持导出清理预览 CSV，CSV 中明确标记 `dry-run` 和 `action_taken=none`。
- 在最大目录/最大文件列表中支持：
  - 复制完整路径
  - 打开目录或文件所在位置
- 默认不跟随符号链接，降低循环扫描风险。
- 遇到无权限目录会记录错误并继续扫描。

## 运行

```bash
cargo run --release
```

首次扫描整个 C 盘可能耗时较长。建议先扫描一个较小目录测试，例如：

```text
C:\Users\你的用户名\Downloads
```

如果扫描目录选错，点击“取消扫描”即可停止后续扫描。取消后界面会显示已经统计到的部分结果。

## 导出与缓存

- 点击“保存结果”会把当前扫描结果保存为 JSON 文件。JSON 会包含完整目录树，可作为之后重新打开的扫描缓存。
- 点击“打开结果”可以加载之前保存的 JSON 文件；加载后会重建内部目录索引，并恢复 Treemap 浏览能力。
- 点击“导出 CSV”会生成单个 CSV 报告，包含概览、目录、当前保留的最大文件、文件类型和访问错误。
- CSV 是面向表格工具的报告格式，不用于恢复扫描缓存；如需恢复结果，请保存 JSON。
- 当前导出功能只写入用户选择的目标文件，不会修改被扫描目录中的任何文件。

## 清理预览

点击“生成清理预览”会基于当前扫描根目录进行一次独立的 dry-run 预览扫描。预览只读取目录项和文件元数据，用于估算潜在可清理空间。

当前版本的清理预览有以下限制和保护：

- 不提供“执行清理”“删除”“移动到回收站”等操作。
- 预计可清理空间只是 dry-run 估算，需要人工判断。
- 命中规则的候选会保留最大的部分用于界面展示，统计总数和总大小仍会持续累计。
- 系统目录和常见用户文档目录中的候选会标记为受保护，并从预计可清理空间中排除。
- 导出的清理预览 CSV 每行都会标记 `mode=dry-run` 和 `action_taken=none`。

## 安全策略

当前版本不会删除、移动或修改任何文件。清理相关功能目前只做 dry-run 预览和报告导出。

后续如果添加真正执行清理的功能，应遵循：

1. 默认 dry-run 预览。
2. 删除前显示预计释放空间。
3. 默认移动到回收站，而不是永久删除。
4. 系统目录和用户文档目录默认保护。
5. 高风险清理规则必须二次确认。
