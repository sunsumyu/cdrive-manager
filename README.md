# C 盘空间管理器

一个用 Rust 编写的桌面 GUI 磁盘空间分析工具，参考了 WinDirStat、QDirStat、Filelight、ncdu、Diskonaut、duc 和 BleachBit 的设计思路。

当前版本是安全优先的 MVP：**只扫描、可视化、缓存和导出 dry-run 报告，不删除、移动或修改被扫描目录中的任何文件**。

## 功能

- 默认扫描 `C:\`，也可以输入其他目录。
- 支持系统目录选择窗口选择扫描目录。
- 开始扫描前会校验路径是否存在、是否是目录。
- 后台线程扫描，界面不会因为扫描 C 盘而完全卡死。
- 支持取消正在进行的扫描，并保留已扫描到的部分结果。
- **扫描过滤配置**：
  - 排除特定目录名（如 `node_modules`、`.git`），支持多个目录用逗号/分号/换行分隔。
  - 排除特定文件扩展名（如 `.tmp`、`.log`），支持多个扩展名。
  - 限制在同一文件系统内扫描，避免跨分区扫描。
- 统计：
  - 累计大小
  - 文件数量
  - 目录数量
  - 访问错误数量
- 显示：
  - 最大目录
  - 最大文件；新扫描/新缓存中，文件搜索会使用完整文件索引
  - 文件扩展名/类型占用
  - 访问错误列表
- Treemap 矩形空间占用图，参考 WinDirStat / QDirStat。
  - 扫描完成或取消后基于完整目录树索引显示目录层级。
  - 支持点击目录块进入子目录视图。
  - **面包屑导航**：显示当前路径层级，快速返回上级或根目录。
  - 支持显示当前目录”直属文件”空间块。
  - 超过显示上限时用”其它”块聚合较小项目，并为目录使用稳定颜色。
  - 支持在目录块右键复制路径或打开位置。
- Filelight 风格旭日图。
  - 可在 Treemap 与旭日图之间切换。
  - 从内到外展示目录层级，并聚合较小项目。
  - 支持进入目录、复制路径和打开位置。
- 支持保存和打开带元信息的 JSON 扫描结果缓存，重新打开后可继续查看完整目录树、完整文件索引和可视化结果。
- 支持 SQLite 扫描缓存管理：
  - 扫描完整完成后会自动把最新结果写入本机应用缓存目录。
  - 输入同一根目录后可点击”打开缓存”直接恢复最近一次扫描。
  - **缓存管理窗口**：查看所有已保存的缓存记录、加载指定缓存、删除缓存记录。
  - SQLite 中保存的是扫描结果 JSON blob，避免对被扫描目录做任何修改。
- 支持导出 CSV 报告，便于用表格工具分析概览、目录、文件、文件类型和错误记录。
- 支持 dry-run 清理预览：
  - 独立后台线程只读取文件元数据，不删除、移动或修改文件。
  - 可取消预览，并保留已发现的候选。
  - 初始保守规则包括临时文件扩展名、日志/转储、备份/旧文件和明显临时/缓存目录中的文件。
  - 规则可在界面中勾选启用/停用；规则变化会要求重新生成 dry-run 预览。
  - 会标记系统目录和常见用户文档目录等受保护候选，受保护候选不计入预计可清理空间。
  - 支持导出清理预览 CSV，CSV 中明确标记 `dry-run` 和 `action_taken=none`。
- 支持 dry-run 重复文件检测：
  - 先按文件大小分组，只对同大小候选读取内容并计算 BLAKE3 哈希。
  - **可配置最小候选大小**：仅对大于指定大小的文件进行哈希比对，减少计算开销。
  - 每组默认保留修改时间最早、路径排序最靠前的一个文件。
  - 其它同哈希文件只作为”重复副本”预览，不执行删除。
  - 受保护重复副本不计入预计可回收空间。
  - 支持取消检测、查看重复组和导出重复文件 CSV；CSV 中同样标记 `dry-run` 和 `action_taken=none`。
- 在目录/文件/清理预览/重复文件列表中支持：
  - 复制完整路径
  - 打开目录或文件所在位置
- 在目录标签中输入关键词时，会搜索完整目录树，并可一键”定位图中”跳转空间占用图。
- 默认不跟随符号链接，降低循环扫描风险。
- 遇到无权限目录会记录错误并继续扫描。

## 运行

```bash
cargo run --release
```

首次扫描整个 C 盘可能耗时较长。建议先扫描一个较小目录测试，例如：

```text
C:\Users\你的用户名\Downloads
```

如果扫描目录选错，点击”取消扫描”即可停止后续扫描。取消后界面会显示已经统计到的部分结果。

## 导出与缓存

- 点击”保存结果”会把当前扫描结果保存为 JSON 文件。JSON 会包含完整目录树、完整文件索引、缓存 schema 版本、保存时间戳和应用版本，可作为之后重新打开的扫描缓存。
- 点击”打开结果”可以加载之前保存的 JSON 文件；加载后会重建内部目录索引，并恢复可视化浏览能力。
- 扫描正常完成后，程序会自动写入 SQLite 最新扫描缓存；点击”打开缓存”可以按当前输入根目录加载最近一次扫描。
- 点击”缓存管理”可以打开缓存管理窗口，查看所有已保存的缓存记录、加载指定缓存或删除缓存记录。
- 点击”导出 CSV”会生成单个 CSV 报告，包含概览、目录、文件、文件类型和访问错误。新扫描/新缓存会导出完整文件索引；旧缓存或扫描中快照可能只包含最大文件列表。
- CSV 是面向表格工具的报告格式，不用于恢复扫描缓存；如需恢复结果，请保存 JSON 或使用 SQLite 最新缓存。
- 当前导出功能只写入用户选择的目标文件，不会修改被扫描目录中的任何文件。

## 清理预览

点击”生成清理预览”会基于当前扫描根目录进行一次独立的 dry-run 预览扫描。预览只读取目录项和文件元数据，用于估算潜在可清理空间。

当前版本的清理预览有以下限制和保护：

- 可在清理预览标签中启用或停用具体规则；规则变化后已有预览会失效，需要重新生成。
- 不提供”执行清理””删除””移动到回收站”等操作。
- 预计可清理空间只是 dry-run 估算，需要人工判断。
- 命中规则的候选会保留最大的部分用于界面展示，统计总数和总大小仍会持续累计。
- 系统目录和常见用户文档目录中的候选会标记为受保护，并从预计可清理空间中排除。
- 导出的清理预览 CSV 每行都会标记 `mode=dry-run` 和 `action_taken=none`。

## 重复文件预览

点击”查找重复文件”会基于当前扫描根目录进行一次独立的 dry-run 重复检测。检测过程只对必要候选读取文件内容并计算哈希，不会修改文件。

当前版本的重复检测有以下限制和保护：

- **可配置最小候选大小**：在重复文件标签中设置最小大小阈值，仅对大于阈值的文件进行哈希计算。
- 只对大小相同且大于最小阈值的文件计算 BLAKE3 哈希，减少不必要读取。
- 相同大小、相同哈希的文件会组成一个重复组。
- 每组显示一个保留文件和若干重复副本；选择保留文件的规则是修改时间优先，其次路径排序。
- 受保护路径会显示在结果中，但不计入预计可回收空间。
- 结果是 dry-run 报告，不提供删除、移动或替换操作。
- 导出的重复文件 CSV 每行都会标记 `mode=dry-run` 和 `action_taken=none`。

## 安全策略

当前版本不会删除、移动或修改被扫描目录中的任何文件。清理和重复文件相关功能目前只做 dry-run 预览和报告导出。

后续如果添加真正执行清理的功能，应遵循：

1. 默认 dry-run 预览。
2. 删除前显示预计释放空间。
3. 默认移动到回收站，而不是永久删除。
4. 系统目录和用户文档目录默认保护。
5. 高风险清理规则必须二次确认。
